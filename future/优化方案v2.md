### 2. 统一错误处理扩展

**现状**: 已实现 `wrapIpcHandler` 和 `ErrorCodes`，但错误翻译未集成

**目标**: 错误码自动翻译为本地语言

```javascript
// src/main/utils/errors.js
const { t } = require('./translations')

// 扩展 handleIpcError，翻译错误消息
export function handleIpcError(error, context) {
  if (error instanceof AppError) {
    return {
      success: false,
      error: t(`errors.${error.code}`) || error.message,
      code: error.code
    }
  }
  // 未知错误翻译
  return {
    success: false,
    error: t('errors.unknown'),
    code: 'UNKNOWN_ERROR'
  }
}
```

## 安全加固

### 1. CSP 强化

**当前问题**:
```html
<!-- index.html -->
<meta http-equiv="Content-Security-Policy" content="
  script-src 'self' 'unsafe-inline' 'unsafe-eval';  <!-- 危险 -->
  style-src 'self' 'unsafe-inline';                  <!-- 危险 -->
">
```

**强化方案**:
```html
<!-- index.html -->
<meta http-equiv="Content-Security-Policy" content="
  default-src 'self';
  script-src 'self';
  style-src 'self';
  connect-src 'self';
  img-src 'self' data:;
  font-src 'self';
">
```

**移除 `'unsafe-inline'` 和 `'unsafe-eval'`**:
- 需要将所有内联脚本移至 `.js` 文件
- 需要将所有内联样式移至 `.less` 文件
- 预计需要 1-2 周工作量

### 2. webSecurity 重新评估

**当前问题**:
```javascript
// main.js:246
webPreferences: {
  webSecurity: false  // 危险
}
```

**分析**:
- `webSecurity: false` 禁用同源策略
- 可能被恶意网页利用
- 保留理由: 开发调试方便

**决策**: 改为 `webSecurity: process.env.NODE_ENV === 'production'`

**完成状态**: ✅ 已完成
- main.js: webSecurity 改为 `process.env.NODE_ENV === 'production'`
- src/main/window.js: webSecurity 改为 `process.env.NODE_ENV === 'production'`

### 3. IPC 输入验证

**问题**: IPC handlers 缺少输入校验

**目标**: 为所有 IPC handler 添加参数验证

```javascript
// src/main/ipc/settings.js
import { validate } from '@/main/utils/validator'

ipcMain.handle('save-settings', wrapIpcHandler(async (event, data) => {
  // 验证 settings 数据结构
  const validation = validateSettings(data)
  if (!validation.valid) {
    return errorResult(validation.errors.join(', '), ErrorCodes.INVALID_INPUT)
  }
  // ...
}))
```

## UI/UX 优化

### 1. 键盘快捷键支持

**现状**: ✅ 已实现 via useKeyboardShortcuts composable

**功能**:
| 快捷键 | 功能 |
|--------|------|
| Ctrl+S | 保存当前设置 |
| Ctrl+, | 打开设置页面 |
| Escape | 关闭对话框 |
| Ctrl+1-6 | 切换导航到对应视图 |

**实现**:
```typescript
// src/renderer/composables/useKeyboardShortcuts.ts
export function useKeyboardShortcuts() {
  const handlers = {
    'ctrl+s': () => saveSettings(),
    'ctrl+,': () => navigateTo('general'),
    'escape': () => closeActiveDialog(),
  }
  
  const handleKeyDown = (e) => {
    const key = `${e.ctrlKey ? 'ctrl+' : ''}${e.key.toLowerCase()}`
    if (handlers[key]) {
      e.preventDefault()
      handlers[key]()
    }
  }
  
  onMounted(() => window.addEventListener('keydown', handleKeyDown))
  onUnmounted(() => window.removeEventListener('keydown', handleKeyDown))
}
```

## 风险与注意事项

### 高风险事项

1. **CSP 强化**
   - 影响: 移除 `'unsafe-inline'` 可能破坏样式
   - 缓解: 逐步测试，提前预留 1 周时间

2. **TypeScript 迁移**
   - 影响: 大规模重构可能引入 bug
   - 缓解: 从 composables 开始，组件最后

### 注意事项

1. **保持向后兼容**
   - IPC API 接口保持不变
   - 配置文件格式保持不变

2. **渐进式改进**
   - 每次只改动一个小功能
   - 充分测试后再合并

## 总结

本优化方案基于代码库全面分析，针对以下五大方向:

1. **架构优化**: 完成 IPC 迁移，标准化 preload API
2. **代码质量**: 统一常量，逐步启用 TypeScript，输入验证
3. **安全加固**: CSP 强化，webSecurity 修复，IPC 验证
4. **测试完善**: Store 测试，IPC 集成测试，E2E 测试
5. **UI/UX 优化**: 快捷键，深色主题细节，空状态

**预计总工作量**: 6-10 周 (取决于团队规模)

**建议**: 采用敏捷开发方式，每两周完成一个 Phase。
