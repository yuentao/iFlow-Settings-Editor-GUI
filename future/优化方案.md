## 概述

### 项目现状

| 维度 | 现状 | 评分 (1-10) |
|------|------|-------------|
| 功能完整性 | MCP 服务器、API 配置、技能管理、命令管理、仪表盘 | 9 |

### 优化目标

1. **模块化**：降低代码耦合，提高可维护性
2. **类型安全**：引入 TypeScript，减少运行时错误
3. **测试完善**：覆盖率达到 80%+，添加 E2E 测试
4. **性能提升**：减少不必要的渲染和 I/O 操作
5. **安全加固**：完善输入验证，增强安全配置

### 2. 渲染进程状态管理

**现状问题**
- `App.vue` 约 700 行，包含大量状态和业务逻辑
- 状态分散在各个组件，通过 props/emits 传递
- 缺少统一的状态管理

**优化方案**

引入 **Pinia** 作为状态管理库：

```
src/renderer/stores/
├── settings.js          # 全局设置状态
├── apiProfiles.js        # API 配置状态
├── skills.js            # 技能状态
├── commands.js          # 命令状态
├── ui.js                # UI 状态 (主题、侧边栏折叠等)
└── index.js             # Store 导出
```

**Store 设计示例**

```typescript
// src/renderer/stores/settings.ts
import { defineStore } from 'pinia'
import { ref, computed, watch } from 'vue'

export const useSettingsStore = defineStore('settings', () => {
  // State
  const settings = ref<Settings>({})
  const isLoading = ref(true)
  const isSaving = ref(false)
  const modified = ref(false)

  // Getters
  const theme = computed(() => settings.value.uiTheme || 'system')
  const language = computed(() => settings.value.language || 'zh-CN')
  const acrylicEnabled = computed(() => settings.value.acrylicEnabled ?? true)
  const acrylicIntensity = computed(() => settings.value.acrylicIntensity ?? 50)

  // Actions
  async function loadSettings() {
    isLoading.value = true
    try {
      const result = await window.electronAPI.loadSettings()
      if (result.success) {
        settings.value = result.data
      }
    } finally {
      isLoading.value = false
    }
  }

  async function saveSettings() {
    isSaving.value = true
    try {
      await window.electronAPI.saveSettings(settings.value)
      modified.value = false
    } finally {
      isSaving.value = false
    }
  }

  // Auto-save with debounce
  let saveTimer: number | null = null
  watch(settings, () => {
    if (!isLoading.value) {
      modified.value = true
      if (saveTimer) clearTimeout(saveTimer)
      saveTimer = setTimeout(saveSettings, 1000) // 1秒防抖
    }
  }, { deep: true })

  return {
    settings, isLoading, isSaving, modified,
    theme, language, acrylicEnabled, acrylicIntensity,
    loadSettings, saveSettings
  }
})
```

```typescript
const apiProfileSchema = Joi.object({
  name: Joi.string().pattern(/^[\w\-]+$/).max(50).required(),
  apiKey: Joi.string().allow('').optional(),
  baseUrl: Joi.string().uri().allow('').optional(),
  modelName: Joi.string().max(100).optional(),

  searchApiKey: Joi.string().allow('').optional(),

const mcpServerSchema = Joi.object({
  id: Joi.string().uuid().required(),
  name: Joi.string().max(100).required(),
  command: Joi.string().not(Joi.ref('name')).required(), // 防止命令注入
  args: Joi.array().items(Joi.string()).optional(),
  env: Joi.object().pattern(Joi.string(), Joi.string()).optional(),
  enabled: Joi.boolean().required()
})

export function validateApiProfile(data: any) {
  return apiProfileSchema.validate(data, { abortEarly: false })

export function validateMcpServer(data: any) {
  return mcpServerSchema.validate(data, { abortEarly: false })
```

## 性能优化

### 1. 防抖保存

**现状问题**
- 每次设置变更立即触发保存
- 可能导致频繁的 I/O 操作

**优化方案**

```typescript
// 使用 lodash.debounce 或自定义实现
import { ref, watch } from 'vue'
import { useDebounceFn } from '@vueuse/core'

export function useAutoSave(settings, saveFunction) {
  const isSaving = ref(false)
  const lastSaved = ref<Date | null>(null)

  const debouncedSave = useDebounceFn(async () => {
    isSaving.value = true
    try {
      await saveFunction()
      lastSaved.value = new Date()
    } finally {
      isSaving.value = false
    }
  }, 1000)

  watch(settings, () => {
    debouncedSave()
  }, { deep: true })

  return { isSaving, lastSaved }
}
```

### 4. 资源缓存

**现状问题**
- 技能列表每次切换都重新请求

**优化方案**

```typescript
// stores/skills.ts
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'

export const useSkillsStore = defineStore('skills', () => {
  const skills = ref<Skill[]>([])
  const lastFetchTime = ref<number | null>(null)
  const CACHE_TTL = 5 * 60 * 1000 // 5分钟缓存

  const isCacheValid = computed(() => {
    if (!lastFetchTime.value) return false
    return Date.now() - lastFetchTime.value < CACHE_TTL
  })

  async function fetchSkills(force = false) {
    if (!force && isCacheValid.value) {
      return skills.value
    }

    const result = await window.electronAPI.listSkills()
    if (result.success) {
      skills.value = result.data
      lastFetchTime.value = Date.now()
    }
    return skills.value
  }

  return { skills, fetchSkills }
})
```

```typescript
// 恶意的命令检查
const DANGEROUS_PATTERNS = [
  /;\s*rm\s+/i,
  /;\s*del\s+/i,
  /\$\(/,
  /`.*`/,
  /\|.*&/,

export function isCommandSafe(command: string): boolean {

// 在 IPC 处理器中使用

})
```

```typescript
// 实现简单的配置版本历史
const BACKUP_DIR = path.join(app.getPath('userData'), 'backups')

async function createBackup(settings) {
  const timestamp = new Date().toISOString().replace(/[:.]/g, '-')
  const backupPath = path.join(BACKUP_DIR, `settings-${timestamp}.json`)

  // 保留最近 10 个备份
  const backups = await fs.readdir(BACKUP_DIR)
  if (backups.length >= 10) {
    const oldest = backups.sort()[0]
    await fs.unlink(path.join(BACKUP_DIR, oldest))
  }

  await fs.writeFile(backupPath, JSON.stringify(settings, null, 2), 'utf-8')
}

ipcMain.handle('list-backups', async () => {
  const backups = await fs.readdir(BACKUP_DIR)
  return backups.map(name => ({
    name,
    path: path.join(BACKUP_DIR, name),
    time: fs.statSync(path.join(BACKUP_DIR, name)).mtime
  }))
})

ipcMain.handle('restore-backup', async (event, backupPath) => {
  const content = await fs.readFile(backupPath, 'utf-8')
  const settings = JSON.parse(content)
  await saveSettings(settings)
})
```

## 风险与注意事项

### 高风险事项

1. **TypeScript 迁移**
   - 风险：大规模重构可能导致功能回归
   - 缓解：分阶段迁移，保持每次提交可运行

2. **状态管理重构**
   - 风险：从 props/emits 切换到 Pinia 可能引入 bug
   - 缓解：先并行运行，逐步移除直接状态传递

3. **组件懒加载**
   - 风险：首次加载慢，影响用户体验
   - 缓解：保留骨架屏，控制懒加载范围

### 注意事项

1. **保持向后兼容**
   - 配置文件格式不变
   - IPC API 接口不变

2. **渐进式改进**
   - 每次只做一个小改动
   - 充分测试后再合并

3. **性能基准测试**
   - 优化前记录性能指标
   - 优化后对比验证
