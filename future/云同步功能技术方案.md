# iFlow Settings Editor - 云同步功能技术方案（MVP - Dropbox/OneDrive 版）

## 📋 项目概述

### 背景
iFlow Settings Editor 作为一款桌面配置管理工具，用户可能在多台设备上使用（如工作电脑、个人电脑）。目前配置数据仅存储在本地 (`~/.iflow/settings.json`)，缺乏跨设备同步能力。

### 目标（MVP）
实现最小可行产品的云同步功能，仅同步核心配置：
- **API 配置**（包含敏感信息需加密）
- **MCP 服务器配置**（可能包含 URL/密钥）

### 核心原则
- **安全第一**：端到端加密，云服务商无法读取明文
- **用户可控**：用户使用自己的云存储账号，可随时撤销授权、删除数据
- **隐私保护**：不收集用户使用习惯等额外信息
- **离线可用**：本地缓存，网络异常时不影响使用
- **轻量简洁**：不同步技能、命令等大文件

## 🏗️ 技术方案选择

### 推荐方案：Dropbox / OneDrive API + 端到端加密

#### 为什么选择云存储 API？
1. **零服务器成本**：无需购买/维护云服务器
2. **用户自主**：用户使用自己的账号授权，数据存在用户自己的云盘
3. **文件同步成熟**：Dropbox/OneDrive 已解决冲突、版本历史等复杂问题
4. **快速上线**：只需开发客户端，无需后端
5. **跨平台**：API 在 Windows/macOS/Linux 上表现一致

#### 支持列表（按优先级）
1. **Dropbox API v2**（推荐首期）
   - 免费版 2GB 空间（配置数据足够）
   - API 限制：每分钟 1000 次调用（足够）
   - 支持文件上传/下载/版本管理
   - OAuth 2.0 流程简单，支持本地回环回调

2. **OneDrive API (Microsoft Graph)**（二期）
   - 免费版 5GB 空间
   - API 限制：每分钟 10000 次调用
   - 与 Windows 深度集成
   - 企业用户可能已有账号

3. **Google Drive**（备选）
   - 免费版 15GB 空间
   - API 配额较严格
   - 国际用户友好

**MVP 阶段**：先实现 **Dropbox**，后续扩展 OneDrive。

## 🔐 安全设计（核心）

### 加密策略：端到端加密（E2EE）

#### 1. 加密流程
```
用户同步密码
    ↓ PBKDF2 (10万次迭代 + 随机盐)
256位 AES 密钥
    ↓ AES-256-GCM (每次生成随机 IV)
加密数据 + 认证标签
    ↓ Base64 编码
保存为 JSON 文件
    ↓
上传到用户自己的 Dropbox/OneDrive
```

#### 2. 加密字段
**必须加密**：
- `apiKey`（所有 API 配置）
- `searchApiKey`
- `cna`
- MCP 服务器的 `apiKey`
- 任何包含 token/密钥的字段

**可明文同步**（不敏感）：
- `baseUrl`（服务器地址）
- `modelName`（模型名称）
- `selectedAuthType`（认证类型）
- MCP 服务器的 `name`、`url`、`description`

#### 3. 密钥管理
**方案：独立同步密码（用户设置）**
- 用户首次启用云同步时设置同步密码（至少 8 位）
- 密码通过 PBKDF2 派生为 AES 密钥，salt 随机生成并存储
- 密码本身不存储，仅存储 salt 和密钥哈希（用于验证）
- 更换设备时需输入同步密码才能解密数据
- **密码无法找回**，忘记则需重置云端数据

│      用户自己的 Dropbox 云存储（文件形式）        │

```

### 数据流

#### 上传流程
```
1. 用户点击「同步」或自动触发
   ↓
2. 读取本地 settings.json
   ↓
3. 提取需要同步的字段：
   - apiProfiles（完整对象）
   - mcpServers（完整数组）
   - currentApiProfile（字符串）
   ↓
4. 加密敏感字段（AES-256-GCM）
   ↓
5. 生成同步文件内容：
   {
     "version": "1.0",
     "timestamp": "2026-04-23T10:30:00Z",
     "deviceId": "设备唯一标识",
     "data": {
       "apiProfiles": {...},
       "mcpServers": [...],
       "currentApiProfile": "default"
     }
   }
   ↓
6. 计算文件哈希（SHA-256，用于冲突检测）
   ↓
7. 使用 Dropbox SDK 上传到用户自己的 Dropbox
   ↓
8. 更新本地元数据（最后同步时间）
   ↓
9. 显示同步成功
```

#### 下载/合并流程
```
1. 用户在新设备上登录 Dropbox 并授权
   ↓
2. 从用户 Dropbox 下载所有设备的配置文件
   ↓
3. 使用同步密码解密每个文件
   ↓
4. 与本地配置合并（冲突处理）
   ↓
5. 写入本地 settings.json
   ↓
6. 通知 UI 刷新
```

## 🔄 同步策略

### 触发时机

#### 场景 1：自动同步开关开启
- **应用启动时**：自动检查并同步云端配置（如果已授权且密码已设置）
- **配置变更后**：防抖 5 秒自动同步（避免频繁上传）

#### 场景 2：自动同步开关关闭
- **仅手动触发**：用户点击「立即同步」按钮才执行同步

#### 场景 3：首次启用
- 用户需先：
  1. 设置同步密码
  2. 授权 Dropbox
  3. 配置自动同步开关（可选）
- 首次同步会上传当前配置到云端

### 同步开关状态机
```
enabled（启用云同步）
  ├─ false：云同步功能完全禁用，不显示子选项
  └─ true：显示配置区域
      ├─ autoSyncEnabled（自动同步）
      │   ├─ false：仅手动同步
      │   └─ true：启动自动同步 + 变更后自动同步
      ├─ hasPassword：是否已设置同步密码
      ├─ isAuthorized：Dropbox 授权状态
      └─ deviceName：设备名称
```

### 冲突解决策略
**基于文件版本**：
- 每个设备上传独立文件（`config-<deviceId>.json.enc`）
- 下载时获取所有设备的最新版本
- 按 `timestamp` 排序，取最新的作为主配置
- 保留所有设备的文件（Dropbox 版本历史）

**合并策略**：
```
1. 下载所有设备的 config 文件
2. 解密每个文件
3. 按 timestamp 降序排列
4. 取最新版本作为基础
5. 合并其他设备的非冲突字段（如新增的 apiProfile）
6. 冲突字段（同一配置项）保留最新的
7. 写入本地并上传新版本
```

### 错误处理
- **OAuth 过期/撤销**：提示用户重新授权，打开浏览器
- **网络失败**：重试 3 次（指数退避），本地队列暂存
- **解密失败**：提示「同步密码错误」，要求重新输入
- **API 限流**：等待 1 分钟后重试
- **文件冲突**：使用 Dropbox 冲突副本（`.conflict` 后缀）

function formatTime(timestamp) {
  return new Date(timestamp).toLocaleString('zh-CN', {
    year: 'numeric',
    month: '2-digit',

function debounce(fn, delay) {
  let timer = null
  return function (...args) {
    if (timer) clearTimeout(timer)
    timer = setTimeout(() => fn.apply(this, args), delay)
  }
}

function getCloudApi(provider) {
  if (provider === 'dropbox') {
    return new DropboxAPI(localSettings.cloudSync.accessToken)
  } else if (provider === 'onedrive') {

function startLocalHttpServer(port) {
  return new Promise((resolve, reject) => {
    const server = require('http').createServer((req, res) => {
      if (req.url.startsWith('/callback')) {
        const url = require('url').parse(req.url, true)
        const code = url.query.code
        const error = url.query.error
        if (error) {
          reject(new Error(`OAuth 失败: ${error}`))
        } else if (code) {
          res.writeHead(200, { 'Content-Type': 'text/html' })
          res.end('<h1>授权成功！请返回应用继续操作。</h1>')
          resolve(code)
        }
      }
    })
    server.listen(port, '127.0.0.1', () => resolve(server))
    server.on('error', reject)
  })
}

    const redirectUri = `http://localhost:${port}/callback`
    const scopes = ['Files.ReadWrite']
    return `https://login.microsoftonline.com/common/oauth2/v2.0/authorize?client_id=${clientId}&response_type=code&redirect_uri=${encodeURIComponent(redirectUri)}&scope=${scopes.join(' ')}&code_challenge=${codeChallenge}&code_challenge_method=S256&state=${state}`
  }
}

function generateCodeVerifier() {
  const array = crypto.getRandomValues(new Uint8Array(32))
  return btoa(String.fromCharCode(...array))
    .replace(/\+/g, '-')
    .replace(/\//g, '_')
    .replace(/=+$/, '')
}

function generateCodeChallengeSync(codeVerifier) {
  const encoder = new TextEncoder()
  const data = encoder.encode(codeVerifier)
  const hash = crypto.subtle.digestSync('SHA-256', data)
  const base64 = btoa(String.fromCharCode(...new Uint8Array(hash)))
  return base64.replace(/\+/g, '-').replace(/\//g, '_').replace(/=+$/, '')
}

function waitForOAuthCallback(server, timeoutMs = 300000) {
  return new Promise((resolve, reject) => {
    const timeout = setTimeout(() => {
      server.close()
      reject(new Error('OAuth 授权超时（5 分钟）'))
    }, timeoutMs)
    server.once('callback', (code) => {
      clearTimeout(timeout)
      resolve(code)
    })
  })
}

async function exchangeCodeForToken(provider, code) {
  const codeVerifier = localSettings.cloudSync.tempCodeVerifier
  if (!codeVerifier) throw new Error('缺少 code_verifier')
  delete localSettings.cloudSync.tempCodeVerifier
  writeSettings(localSettings)

  if (provider === 'dropbox') {
    const clientId = 'YOUR_DROPBOX_APP_KEY'
    const clientSecret = 'YOUR_DROPBOX_APP_SECRET'
    const redirectUri = `http://localhost:${oauthPort.value}/callback`
    const response = await fetch('https://api.dropbox.com/oauth2/token', {
      method: 'POST',
      headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
      body: new URLSearchParams({
        code, grant_type: 'authorization_code', client_id: clientId,
        client_secret: clientSecret, redirect_uri: redirectUri, code_verifier: codeVerifier
      })
    })
    if (!response.ok) throw new Error(`Token 交换失败: ${response.statusText}`)
    const data = await response.json()
    return { accessToken: data.access_token, refreshToken: data.refresh_token, expiry: Date.now() + data.expires_in * 1000 }
  } else if (provider === 'onedrive') {
    const clientId = 'YOUR_ONEDRIVE_CLIENT_ID'
    const clientSecret = 'YOUR_ONEDRIVE_CLIENT_SECRET'
    const redirectUri = `http://localhost:${oauthPort.value}/callback`
    const response = await fetch('https://login.microsoftonline.com/common/oauth2/v2.0/token', {
      method: 'POST',
      headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
      body: new URLSearchParams({
        code, grant_type: 'authorization_code', client_id: clientId,
        client_secret: clientSecret, redirect_uri: redirectUri,
        code_verifier: codeVerifier, scope: 'Files.ReadWrite'
      })
    })
    if (!response.ok) throw new Error(`Token 交换失败: ${response.statusText}`)
    const data = await response.json()
    return { accessToken: data.access_token, refreshToken: data.refresh_token, expiry: Date.now() + data.expires_in * 1000 }
  }
}

## 🔄 OAuth 2.0 流程（用户账号授权）

### Dropbox OAuth 流程
```
1. 用户点击「授权 Dropbox」
   ↓
2. 应用生成授权 URL（使用 Dropbox App Key）
   ↓
3. 打开系统浏览器，用户登录 Dropbox 账号
   ↓
4. 用户授权应用访问 /Apps/iFlow-Settings-Editor/ 文件夹
   ↓
5. Dropbox 重定向到本地回环地址
   http://localhost:PORT/callback?code=AUTH_CODE
   ↓
6. 本地 HTTP 服务器捕获回调，提取 code
   ↓
7. 用 code 换取 access_token 和 refresh_token
   ↓
8. 将 token 加密保存到本地配置
   ↓
9. 授权完成，可开始同步
```

### 关键点
- **App Key/Secret**：在 Dropbox 开发者平台注册应用，获取 App Key
- **回调地址**：`http://localhost:53682/callback`（随机端口）
- **Token 存储**：access_token 和 refresh_token 加密保存到 `settings.json` 的 `cloudSync` 段
- **Token 刷新**：access_token 过期后使用 refresh_token 自动刷新
- **用户可撤销**：用户可在 Dropbox 设置中撤销授权，应用检测到后提示重新授权

### 云存储切换与重新授权

#### 切换云存储服务（`onCloudProviderChange`）
当用户从 Dropbox 切换到 OneDrive（或反之）：
1. **清空当前授权信息**：删除 `accessToken`、`refreshToken`、`tokenExpiry`
2. **重置授权状态**：`isAuthorized = false`
3. **显示提示**：`"已切换至 {{providerName}}，请重新授权"`
4. **保留已设置的同步密码**：密码不随云存储切换而清除
5. **不删除云端数据**：用户原有的 Dropbox/OneDrive 数据保留，需手动清理

```javascript

#### 重新授权（`reauthorize`）
用户点击「重新授权」按钮时：
1. 与首次授权流程相同（打开浏览器，OAuth 流程）
2. 新的 token 会覆盖旧的 token
3. 授权成功后提示：`"重新授权成功"`

#### 取消授权（`revokeAuth`）
用户点击「取消授权」按钮时：
1. 本地删除 `accessToken`、`refreshToken`
2. 设置 `isAuthorized = false`
3. **不清除云端数据**：用户 Dropbox 中的配置文件保留
4. 提示：`"已取消授权，云端数据仍保留在您的 Dropbox 中"`

## 🔄 Dropbox API 集成细节

### OAuth 2.0 完整流程
```javascript
const { Dropbox } = require('dropbox')
const open = require('open')
const http = require('http')
const url = require('url')
const crypto = require('crypto')

class DropboxOAuth {
  constructor(appKey, appSecret) {
    this.dbx = new Dropbox({ clientId: appKey, clientSecret: appSecret })
    this.redirectUri = 'http://localhost:53682/callback'
    this.codeVerifier = this.generateCodeVerifier()
  }

  // 1. 生成授权 URL
  getAuthUrl() {
    return this.dbx.getAuthenticationUrl(
      this.redirectUri,
      'code',
      'offline',
      'none',
      this.codeVerifier
    )
  }

  // 2. 打开浏览器
  async startAuth() {
    const authUrl = this.getAuthUrl()
    await open(authUrl)

    // 3. 启动本地服务器等待回调
    return new Promise((resolve, reject) => {
      const server = http.createServer(async (req, res) => {
        const params = new URL(req.url, this.redirectUri).searchParams
        const code = params.get('code')
        const error = params.get('error')

        if (error) {
          reject(new Error(error))
        } else {
          try {
            // 4. 交换 token
            const tokenResponse = await this.dbx.getAccessTokenFromCode(
              code,
              this.redirectUri,
              this.codeVerifier
            )
            resolve({
              accessToken: tokenResponse.result.access_token,
              refreshToken: tokenResponse.result.refresh_token,
              expiresIn: tokenResponse.result.expires_in
            })
          } catch (err) {
            reject(err)
          }
        }
        res.end('授权成功，可关闭此窗口')
        server.close()
      })

      server.listen(53682, () => {
        console.log('OAuth 服务器监听中...')
      })
    })
  }

  // 5. 刷新 token
  async refreshToken(refreshToken) {
    const response = await this.dbx.refreshAccessToken(refreshToken)
    return response.result
  }

  generateCodeVerifier() {
    return crypto.randomBytes(32).toString('hex')
  }
}

### 核心类设计

#### 1. CryptoManager（加密模块）
**文件**：`src/main/crypto.js`

```javascript
const crypto = require('crypto')

class CryptoManager {
  constructor() {
    this.saltLength = 16  // 128位
    this.ivLength = 12    // GCM 推荐 96位
    this.keyLength = 32   // 256位
    this.iterations = 100000  // PBKDF2 迭代次数
  }

  // 从密码派生密钥（PBKDF2）
  async deriveKey(password, salt) {
    return new Promise((resolve, reject) => {
      crypto.pbkdf2(
        password,
        salt,
        this.iterations,
        this.keyLength,
        'sha256',
        (err, derivedKey) => {
          if (err) reject(err)
          else resolve(derivedKey)
        }
      )
    })
  }

  // 加密配置对象
  async encryptConfig(data, password) {
    const salt = crypto.randomBytes(this.saltLength)
    const key = await this.deriveKey(password, salt)

    const iv = crypto.randomBytes(this.ivLength)
    const cipher = crypto.createCipheriv('aes-256-gcm', key, iv)

    const plaintext = JSON.stringify(data)
    let encrypted = cipher.update(plaintext, 'utf8', 'base64')
    encrypted += cipher.final('base64')

    const authTag = cipher.getAuthTag().toString('base64')

    return {
      ciphertext: encrypted,
      iv: iv.toString('base64'),
      authTag,
      salt: salt.toString('base64')
    }
  }

  // 解密配置
  async decryptConfig(encrypted, password) {
    const { ciphertext, iv, authTag, salt } = encrypted

    const key = await this.deriveKey(
      password,
      Buffer.from(salt, 'base64')
    )

    const decipher = crypto.createDecipheriv(
      'aes-256-gcm',
      key,
      Buffer.from(iv, 'base64')
    )
    decipher.setAuthTag(Buffer.from(authTag, 'base64'))

    let decrypted = decipher.update(ciphertext, 'base64', 'utf8')
    decrypted += decipher.final('utf8')

    return JSON.parse(decrypted)
  }

  // 验证密码
  async verifyPassword(password, salt, correctHash) {
    const key = await this.deriveKey(password, Buffer.from(salt, 'base64'))
    const hash = crypto.createHash('sha256').update(key).digest('hex')
    return hash === correctHash
  }

  // 计算配置哈希（用于冲突检测）
  hashConfig(data) {
    const str = JSON.stringify(data, Object.keys(data).sort())
    return crypto.createHash('sha256').update(str).digest('hex')
  }
}

module.exports = CryptoManager
```

#### 2. DropboxOAuth（OAuth 模块）
**文件**：`src/main/services/DropboxOAuth.js`

```javascript
const { Dropbox } = require('dropbox')
const open = require('open')
const http = require('http')
const crypto = require('crypto')
const path = require('path')

class DropboxOAuth {
  constructor(config) {
    this.appKey = config.appKey           // Dropbox App Key
    this.appSecret = config.appSecret     // Dropbox App Secret（可选，PKCE 模式不需要）
    this.redirectUri = config.redirectUri || 'http://localhost:53682/callback'
    this.tokenPath = config.tokenPath     // token 存储路径
    this.codeVerifier = this.generateCodeVerifier()
  }

  // 生成 PKCE code verifier
  generateCodeVerifier() {
    return crypto.randomBytes(32).toString('base64url')
  }

  // 生成授权 URL
  getAuthUrl() {
    const dbx = new Dropbox({ clientId: this.appKey })
    return dbx.getAuthenticationUrl(
      this.redirectUri,
      'code',
      'offline',    // 请求 refresh_token
      'none',       // token 在响应中返回
      this.codeVerifier  // PKCE
    )
  }

  // 启动 OAuth 流程
  async authorize() {
    const authUrl = this.getAuthUrl()

    // 打开浏览器
    await open(authUrl)

    // 等待回调
    return new Promise((resolve, reject) => {
      const server = http.createServer(async (req, res) => {
        const params = new URL(req.url, this.redirectUri).searchParams
        const code = params.get('code')
        const error = params.get('error')

        res.writeHead(200, { 'Content-Type': 'text/html' })
        res.end('<h1>授权成功！请返回应用继续。</h1>')

        if (error) {
          reject(new Error(`OAuth 失败: ${error}`))
        } else {
          try {
            const tokenResponse = await this.exchangeCode(code)
            resolve(tokenResponse)
          } catch (err) {
            reject(err)
          }
        }
        server.close()
      })

      server.listen(53682, () => {
        console.log('OAuth 服务器启动: http://localhost:53682')
      })
    })
  }

  // 交换授权码为 token
  async exchangeCode(code) {
    const dbx = new Dropbox({
      clientId: this.appKey,
      clientSecret: this.appSecret
    })

    const response = await dbx.getAccessTokenFromCode(
      code,
      this.redirectUri,
      this.codeVerifier
    )

    return {
      accessToken: response.result.access_token,
      refreshToken: response.result.refresh_token,
      expiresIn: response.result.expires_in,
      tokenType: response.result.token_type
    }
  }

  // 刷新 token
  async refreshToken(refreshToken) {
    const dbx = new Dropbox({
      clientId: this.appKey,
      clientSecret: this.appSecret
    })

    const response = await dbx.refreshAccessToken(refreshToken)
    return {
      accessToken: response.result.access_token,
      refreshToken: response.result.refresh_token,  // 新的 refresh_token
      expiresIn: response.result.expires_in
    }
  }

#### 3. SyncService（同步服务）
**文件**：`src/main/services/SyncService.js`

```javascript
const CryptoManager = require('../crypto')
const DropboxOAuth = require('./DropboxOAuth')
const { readSettings, writeSettings } = require('../settings')

class SyncService {
  constructor(config) {
    this.config = config
    this.crypto = new CryptoManager()
    this.dropbox = null
    this.isOnline = true
  }

  // 初始化（授权后调用）
  async initialize(accessToken) {
    const { Dropbox } = require('dropbox')
    this.dropbox = new Dropbox({ accessToken })
  }

  // 全量同步
  async sync() {
    if (!this.dropbox) throw new Error('未授权 Dropbox')

    try {
      // 1. 准备本地数据
      const localData = this.prepareSyncData()

      // 2. 加密
      const encrypted = await this.crypto.encryptConfig(
        localData,
        this.config.syncPassword
      )

      // 3. 构造同步文件
      const syncFile = {
        version: '1.0',
        timestamp: new Date().toISOString(),
        deviceId: this.config.deviceId,
        deviceName: this.config.deviceName,
        dataHash: this.crypto.hashConfig(localData),
        encrypted
      }

      // 4. 上传
      const fileName = `config-${this.config.deviceId}.json.enc`
      await this.uploadFile(fileName, JSON.stringify(syncFile))

      // 5. 下载其他设备配置
      const remoteFiles = await this.listConfigFiles()
      const remoteConfigs = []

      for (const file of remoteFiles) {
        if (file.name === fileName) continue  // 跳过自己

        const content = await this.downloadFile(file.name)
        const config = JSON.parse(content)

        // 解密
        try {
          const decrypted = await this.crypto.decryptConfig(
            config.encrypted,
            this.config.syncPassword
          )
          remoteConfigs.push({
            ...config,
            data: decrypted
          })
        } catch (err) {
          console.warn(`解密失败 ${file.name}:`, err.message)
          continue
        }
      }

      // 6. 合并配置
      if (remoteConfigs.length > 0) {
        await this.mergeRemoteConfigs(remoteConfigs)
      }

      // 7. 更新元数据
      this.updateMetadata()

      return { success: true }
    } catch (error) {
      console.error('同步失败:', error)
      return { success: false, error: error.message }
    }
  }

  // 准备同步数据
  prepareSyncData() {
    const settings = readSettings() || {}

    return {
      apiProfiles: settings.apiProfiles || {},
      mcpServers: settings.mcpServers || [],
      currentApiProfile: settings.currentApiProfile || 'default',
      version: settings.cloudSync?.syncVersion || 1
    }
  }

  // 上传文件
  async uploadFile(fileName, content) {
    const filePath = `/Apps/iFlow-Settings-Editor/${fileName}`
    await this.dropbox.filesUpload({
      path: filePath,
      contents: content,
      mode: 'overwrite'
    })
  }

  // 下载文件
  async downloadFile(fileName) {
    const filePath = `/Apps/iFlow-Settings-Editor/${fileName}`
    const response = await this.dropbox.filesDownload({ path: filePath })
    return Buffer.from(response.result.fileBinary, 'base64').toString()
  }

  // 列出配置文件
  async listConfigFiles() {
    const result = await this.dropbox.filesListFolder({
      path: '/Apps/iFlow-Settings-Editor'
    })

    return result.result.entries.filter(
      entry => entry['.tag'] === 'file' && entry.name.endsWith('.json.enc')
    )
  }

  // 合并远程配置
  async mergeRemoteConfigs(remoteConfigs) {
    const localSettings = readSettings() || {}

    // 按时间戳排序（最新的优先）
    remoteConfigs.sort((a, b) =>
      new Date(b.timestamp) - new Date(a.timestamp)
    )

    // 取最新版本作为基础
    const latest = remoteConfigs[0]
    localSettings.apiProfiles = latest.data.apiProfiles
    localSettings.mcpServers = latest.data.mcpServers
    localSettings.currentApiProfile = latest.data.currentApiProfile

    // 合并其他设备的增量（去重）
    for (let i = 1; i < remoteConfigs.length; i++) {
      const cfg = remoteConfigs[i]
      Object.assign(localSettings.apiProfiles, cfg.data.apiProfiles)
      this.mergeMcpServers(localSettings.mcpServers, cfg.data.mcpServers)
    }

    writeSettings(localSettings)
  }

  // 合并 MCP 服务器（去重）
  mergeMcpServers(local, remote) {
    const localMap = new Map(local.map(s => [s.name, s]))
    remote.forEach(s => {
      if (!localMap.has(s.name)) {
        local.push(s)
        localMap.set(s.name, s)
      }
    })
  }

  // 更新元数据
  updateMetadata() {
    const meta = {
      deviceId: this.config.deviceId,
      lastSyncAt: new Date().toISOString(),
      localConfigHash: this.crypto.hashConfig(this.prepareSyncData())
    }
    // 保存到 .sync-metadata.json（加密）
  }

  // 清空云端数据
  async clearCloud() {
    const files = await this.listConfigFiles()
    for (const file of files) {
      await this.dropbox.filesDeleteV2({ path: file.path_lower })
    }
  }
}

module.exports = SyncService

## 🔄 同步触发逻辑总结

### 状态定义
| 状态 | 类型 | 说明 |
|------|------|------|
| `enabled` | 布尔 | 云同步总开关，`false` 时禁用所有同步功能 |
| `autoSyncEnabled` | 布尔 | 自动同步开关，仅在 `enabled=true` 时生效 |
| `hasPassword` | 布尔 | 是否已设置同步密码 |
| `isAuthorized` | 布尔 | Dropbox OAuth 授权状态 |

### 触发条件矩阵

| 场景 | enabled | autoSyncEnabled | 应用启动 | 配置变更 | 手动点击 |
|------|---------|-----------------|----------|----------|----------|
| 云同步禁用 | false | - | ❌ 不触发 | ❌ 不触发 | ❌ 不触发 |
| 自动同步关闭 | true | false | ❌ 不触发 | ❌ 不触发 | ✅ 触发 |
| 自动同步开启 | true | true | ✅ 触发 | ✅ 触发（防抖 5 秒） | ✅ 触发 |

### 关键逻辑
```javascript
// 应用启动时
if (settings.cloudSync?.enabled && settings.cloudSync?.autoSyncEnabled) {
  if (hasPassword && isAuthorized) {
    syncService.sync()  // 自动同步
  }
}

// 配置变更监听
watch(settings, () => {
  if (enabled && autoSyncEnabled) {
    debounce(sync, 5000)  // 防抖 5 秒
  }
}, { deep: true })

// 立即同步按钮
function syncNow() {
  if (!enabled) return
  if (!hasPassword) return showSetPasswordDialog()
  if (!isAuthorized) return startOAuth()
  syncService.sync()  // 手动同步
}
