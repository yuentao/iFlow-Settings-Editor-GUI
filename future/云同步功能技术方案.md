# iFlow Settings Editor - 云同步功能技术方案 v2

> 版本：v2.0
> 日期：2026-04-25
> 基于 v1 方案重写，针对项目实际架构重新设计

---

## 一、v1 方案问题回顾

| 问题 | 说明 |
|------|------|
| 架构脱节 | 渲染进程直接调用 `crypto.subtle`、`require('os')`、`require('http')` 等 Node.js API，在 Electron 渲染进程中不可用 |
| 安全隐患 | OAuth code_verifier、access_token 等敏感数据直接暴露在渲染进程 |
| 同步逻辑放错位置 | 大量同步逻辑写在 Vue 组件 `<script setup>` 中，应归入主进程 |
| 冲突策略过于简单 | "取最新覆盖"会丢失其他设备的新增配置，需更精细的合并 |
| Dropbox 优先策略不适用 | 国内用户访问 Dropbox 受限，应优先支持 WebDAV/OneDrive |
| 缺少 IPC 设计 | 未设计 IPC 通道，与现有模块化 IPC 架构不匹配 |
| 缺少 i18n | 新增 UI 无多语言支持 |
| 整文件加密冗余 | 方案中对整个 data 做加密，实际只有部分字段是敏感的 |

---

## 二、核心设计原则

1. **主进程负责安全操作**：加密、OAuth、网络请求全部在主进程完成
2. **渲染进程只负责 UI**：通过 IPC 调用主进程能力，不直接处理任何敏感数据
3. **与现有架构一致**：复用 `src/main/ipc/` 模式、`preload.js` 桥接、Pinia store 状态管理
4. **国内友好**：首选 WebDAV（坚果云等），其次 OneDrive，Dropbox 作为备选
5. **字段级加密**：只加密真正敏感的字段（apiKey、searchApiKey、cna、env 中的密钥），非敏感字段明文传输方便调试和冲突合并
6. **增量合并优先**：多设备并发修改时，按配置级别（单个 apiProfile / 单个 mcpServer）合并，而非整文件覆盖

### 适配器优先级

| 优先级 | Provider | 说明 |
|--------|----------|------|
| P0 | WebDAV | 坚果云、NextCloud、Synology 等均支持，国内友好，无需注册开发者应用 |
| P1 | OneDrive | Windows 深度集成，需 Azure AD 应用注册 |
| P2 | Dropbox | 国际用户，需 Dropbox 开发者应用 |

### WebDAV 为什么是首选

- **零审批成本**：不需要在云平台注册应用、等待审核
- **用户自带服务**：用户输入自己的 WebDAV 服务器地址 + 用户名 + 应用密码即可
- **国内直达**：坚果云、阿里云盘（WebDAV）、自建 NAS 等均可
- **协议简单**：HTTP PUT/GET/PROPFIND/DELETE，无需 SDK
- **OAuth 免除**：使用 Basic Auth 或 Digest Auth，无需本地回调服务器

## 五、安全设计

### 加密策略：字段级加密

```
同步密码 (用户设置)
    ↓ PBKDF2-SHA256 (600,000 次迭代 + 16 字节随机盐)
256 位 AES 密钥
    ↓ AES-256-GCM (12 字节随机 IV)
加密字段值 → Base64
```

**为什么字段级而非文件级加密**：
- 非敏感字段（baseUrl、modelName、description）明文可读，方便调试
- 合并时可以按字段粒度比较，避免不必要的冲突
- 云端文件人可读部分内容，用户可自行检查数据完整性

### 加密数据格式

```json
{
  "version": 2,
  "timestamp": "2026-04-25T10:30:00Z",
  "deviceId": "abc123",
  "deviceName": "工作电脑",
  "apiProfiles": {
    "default": {
      "selectedAuthType": "openai-compatible",
      "apiKey": "$enc:AQIDBAUG...:iv:authTag",
      "baseUrl": "https://api.example.com",
      "modelName": "gpt-4",
      "searchApiKey": "$enc:BQYGCg...:iv:authTag",
      "cna": "$enc:true:iv:authTag"
    }
  },
  "currentApiProfile": "default",
  "mcpServers": {
    "my-server": {
      "command": "npx",
      "args": ["-y", "some-package"],
      "description": "My MCP Server",
      "env": {
        "API_KEY": "$enc:sk-xxx:iv:authTag"
      }
    }
  },
  "apiProfilesOrder": ["default"]
}
```

- 加密字段以 `$enc:` 前缀标识，后跟 `ciphertext:iv:authTag`（均为 Base64）
- 非敏感字段明文存储
- 解密时识别 `$enc:` 前缀并解密，否则直接使用

### 密钥管理

- 用户首次启用同步时设置同步密码（≥8 位）
- PBKDF2 盐随加密文件保存，密钥哈希保存于本地用于密码验证
- 密码不存储，忘记需重置云端数据
- 每个加密字段独立 IV，防止密文关联分析

## 八、核心类设计

### 1. CryptoManager

```javascript
// src/main/crypto/CryptoManager.js

const crypto = require('crypto')

class CryptoManager {
  constructor() {
    this.saltLength = 16    // 128 位
    this.ivLength = 12      // GCM 推荐 96 位
    this.keyLength = 32     // 256 位
    this.iterations = 600000 // OWASP 2023 推荐的 PBKDF2-SHA256 最小迭代次数
  }

  /** 从密码派生 AES 密钥 */
  deriveKey(password, salt) {
    return crypto.pbkdf2Sync(password, salt, this.iterations, this.keyLength, 'sha256')
  }

  /** 加密单个字段值，返回 `$enc:ciphertext:iv:authTag` 格式 */
  encryptField(plaintext, key) {
    const iv = crypto.randomBytes(this.ivLength)
    const cipher = crypto.createCipheriv('aes-256-gcm', key, iv)
    const encrypted = Buffer.concat([
      cipher.update(plaintext, 'utf8'),
      cipher.final()
    ])
    const authTag = cipher.getAuthTag()
    return `$enc:${encrypted.toString('base64')}:${iv.toString('base64')}:${authTag.toString('base64')}`
  }

  /** 解密 `$enc:...` 格式的字段 */
  decryptField(encString, key) {
    if (!encString.startsWith('$enc:')) return encString
    const parts = encString.split(':')
    // $enc:ciphertext:iv:authTag
    const ciphertext = Buffer.from(parts[1], 'base64')
    const iv = Buffer.from(parts[2], 'base64')
    const authTag = Buffer.from(parts[3], 'base64')
    const decipher = crypto.createDecipheriv('aes-256-gcm', key, iv)
    decipher.setAuthTag(authTag)
    const decrypted = Buffer.concat([
      decipher.update(ciphertext),
      decipher.final()
    ])
    return decrypted.toString('utf8')
  }

  /** 加密整个同步数据（字段级） */
  encryptSyncData(data, password) {
    const salt = crypto.randomBytes(this.saltLength)
    const key = this.deriveKey(password, salt)
    const encrypted = this._encryptFields(data, key)
    encrypted._salt = salt.toString('base64')
    return encrypted
  }

  /** 解密同步数据 */
  decryptSyncData(data, password) {
    const salt = Buffer.from(data._salt, 'base64')
    const key = this.deriveKey(password, salt)
    return this._decryptFields(data, key)
  }

  /** 递归加密敏感字段 */
  _encryptFields(obj, key) {
    const SENSITIVE_KEYS = new Set(['apiKey', 'searchApiKey', 'cna'])
    if (Array.isArray(obj)) return obj.map(item => this._encryptFields(item, key))
    if (obj && typeof obj === 'object') {
      const result = {}
      for (const [k, v] of Object.entries(obj)) {
        if (k === '_salt') continue
        if (k === 'env' && v && typeof v === 'object') {
          // env 对象中的所有值都加密
          result[k] = {}
          for (const [ek, ev] of Object.entries(v)) {
            result[k][ek] = this.encryptField(String(ev), key)
          }
        } else if (SENSITIVE_KEYS.has(k) && typeof v === 'string') {
          result[k] = this.encryptField(v, key)
        } else if (k === 'cna' && typeof v === 'boolean') {
          result[k] = this.encryptField(String(v), key)
        } else if (v && typeof v === 'object') {
          result[k] = this._encryptFields(v, key)
        } else {
          result[k] = v
        }
      }
      return result
    }
    return obj
  }

  /** 递归解密 $enc: 字段 */
  _decryptFields(obj, key) {
    if (Array.isArray(obj)) return obj.map(item => this._decryptFields(item, key))
    if (obj && typeof obj === 'object') {
      const result = {}
      for (const [k, v] of Object.entries(obj)) {
        if (k === '_salt') continue
        if (k === 'env' && v && typeof v === 'object') {
          result[k] = {}
          for (const [ek, ev] of Object.entries(v)) {
            result[k][ek] = typeof ev === 'string' ? this.decryptField(ev, key) : ev
          }
        } else if (typeof v === 'string' && v.startsWith('$enc:')) {
          const decrypted = this.decryptField(v, key)
          // 恢复原始类型
          if (k === 'cna') {
            result[k] = decrypted === 'true'
          } else {
            result[k] = decrypted
          }
        } else if (v && typeof v === 'object') {
          result[k] = this._decryptFields(v, key)
        } else {
          result[k] = v
        }
      }
      return result
    }
    return obj
  }

  /** 生成密码验证哈希 */
  hashKey(key) {
    return crypto.createHash('sha256').update(key).digest('hex')
  }

  /** 计算数据指纹（用于变更检测，不含 _salt） */
  fingerprint(data) {
    const clone = { ...data }
    delete clone._salt
    const str = JSON.stringify(clone, Object.keys(clone).sort())
    return crypto.createHash('sha256').update(str).digest('hex')
  }
}

module.exports = CryptoManager
```

### 2. WebDAVProvider

```javascript
// src/main/services/cloud/WebDAVProvider.js

const https = require('https')
const http = require('http')

class WebDAVProvider {
  /**
   * @param {object} config
   * @param {string} config.serverUrl - WebDAV 服务器地址，如 https://dav.jianguoyun.com/dav/
   * @param {string} config.username
   * @param {string} config.password - 应用专用密码
   * @param {string} config.baseDir - 远程基础目录，默认 /iFlow-Settings/
   */
  constructor(config) {
    this.serverUrl = config.serverUrl.replace(/\/+$/, '')
    this.username = config.username
    this.password = config.password
    this.baseDir = config.baseDir || '/iFlow-Settings/'
  }

  async upload(remotePath, content) {
    const url = `${this.serverUrl}${this.baseDir}${remotePath}`
    return this._request('PUT', url, content)
  }

  async download(remotePath) {
    const url = `${this.serverUrl}${this.baseDir}${remotePath}`
    return this._request('GET', url)
  }

  async list(remoteDir) {
    const url = `${this.serverUrl}${this.baseDir}${remoteDir}`
    const body = `<?xml version="1.0" encoding="utf-8"?>
<d:propfind xmlns:d="DAV:">
  <d:prop><d:getlastmodified/><d:getcontentlength/></d:prop>
</d:propfind>`
    const response = await this._request('PROPFIND', url, Buffer.from(body), { Depth: '1' })
    return this._parsePropfindResponse(response.toString('utf8'))
  }

  async delete(remotePath) {
    const url = `${this.serverUrl}${this.baseDir}${remotePath}`
    return this._request('DELETE', url)
  }

  async isAuthorized() {
    try {
      await this._request('PROPFIND', `${this.serverUrl}${this.baseDir}`, Buffer.from(''), { Depth: '0' })
      return true
    } catch {
      return false
    }
  }

  async authorize() {
    // WebDAV 使用 Basic Auth，无需 OAuth 流程
    // "授权" 即验证凭据是否有效
    const valid = await this.isAuthorized()
    if (!valid) throw new Error('WebDAV 认证失败，请检查用户名和密码')
    return { accessToken: Buffer.from(`${this.username}:${this.password}`).toString('base64') }
  }

  async refreshToken() {
    // WebDAV 无需刷新 token
    return this.authorize()
  }

  async revoke() {
    // WebDAV 无需撤销，清除本地凭据即可
  }

  dispose() {}

  _request(method, url, body, extraHeaders = {}) {
    return new Promise((resolve, reject) => {
      const parsedUrl = new URL(url)
      const mod = parsedUrl.protocol === 'https:' ? https : http
      const auth = Buffer.from(`${this.username}:${this.password}`).toString('base64')

      const headers = {
        'Authorization': `Basic ${auth}`,
        ...extraHeaders,
      }
      if (body && body.length > 0) {
        headers['Content-Type'] = method === 'PROPFIND' ? 'application/xml' : 'application/octet-stream'
        headers['Content-Length'] = body.length
      }

      const req = mod.request(url, { method, headers }, (res) => {
        const chunks = []
        res.on('data', chunk => chunks.push(chunk))
        res.on('end', () => {
          const buf = Buffer.concat(chunks)
          if (res.statusCode >= 200 && res.statusCode < 300) {
            resolve(buf)
          } else if (res.statusCode === 401) {
            reject(new Error('WEBDAV_AUTH_FAILED'))
          } else if (res.statusCode === 404) {
            reject(new Error('WEBDAV_NOT_FOUND'))
          } else {
            reject(new Error(`WEBDAV_ERROR_${res.statusCode}`))
          }
        })
      })
      req.on('error', reject)
      if (body && body.length > 0) req.write(body)
      req.end()
    })
  }

  _parsePropfindResponse(xml) {
    const files = []
    const hrefRegex = /<d:href>([^<]+)<\/d:href>/g
    const lmRegex = /<d:getlastmodified>([^<]+)<\/d:getlastmodified>/g
    const sizeRegex = /<d:getcontentlength>([^<]+)<\/d:getcontentlength>/g

    const hrefs = [...xml.matchAll(hrefRegex)].map(m => m[1])
    const lms = [...xml.matchAll(lmRegex)].map(m => m[1])
    const sizes = [...xml.matchAll(sizeRegex)].map(m => parseInt(m[1], 10))

    // 跳过第一个（目录自身）
    for (let i = 1; i < hrefs.length; i++) {
      const name = decodeURIComponent(hrefs[i]).split('/').filter(Boolean).pop()
      if (!name) continue
      files.push({
        name,
        path: hrefs[i],
        lastModified: lms[i - 1] || new Date().toISOString(),
```

### 3. SyncService

```javascript
// src/main/services/SyncService.js

const CryptoManager = require('../crypto/CryptoManager')
const { readSettings, writeSettings } = require('./configService')
const { logger } = require('../utils/logger')
const os = require('os')
const crypto = require('crypto')

class SyncService {
  constructor() {
    this.crypto = new CryptoManager()
    this.provider = null
    this.isSyncing = false
    this.deviceId = this._getOrCreateDeviceId()
  }

  /** 设置云存储适配器 */
  setProvider(provider) {
    this.provider = provider
  }

  /** 获取同步状态 */
  getStatus() {
    const settings = readSettings() || {}
    const cs = settings.cloudSync || {}
    return {
      enabled: cs.enabled || false,
      autoSyncEnabled: cs.autoSyncEnabled || false,
      hasPassword: !!cs.passwordHash,
      isAuthorized: !!cs.providerConfig,
      provider: cs.provider || null,
      deviceName: cs.deviceName || os.hostname(),
      deviceId: this.deviceId,
      lastSyncAt: cs.lastSyncAt || null,
      lastSyncError: cs.lastSyncError || null,
      isSyncing: this.isSyncing,
    }
  }

  /** 推送本地配置到云端 */
  async push(password) {
    if (!this.provider) throw new Error('未配置云存储服务')
    if (this.isSyncing) throw new Error('正在同步中')

    this.isSyncing = true
    try {
      const settings = readSettings() || {}
      const syncData = this._extractSyncData(settings)
      const encrypted = this.crypto.encryptSyncData(syncData, password)

      const fileContent = Buffer.from(JSON.stringify({
        version: 2,
        timestamp: new Date().toISOString(),
        deviceId: this.deviceId,
        deviceName: (settings.cloudSync || {}).deviceName || os.hostname(),
        fingerprint: this.crypto.fingerprint(encrypted),
        data: encrypted,
      }, null, 2))

      await this.provider.upload(`devices/config-${this.deviceId}.json`, fileContent)

      // 更新元数据
      settings.cloudSync = settings.cloudSync || {}
      settings.cloudSync.lastSyncAt = new Date().toISOString()
      settings.cloudSync.lastSyncError = null
      writeSettings(settings)

      logger.info('[CloudSync] Push succeeded')
      return { success: true }
    } catch (error) {
      logger.error('[CloudSync] Push failed:', error)
      const settings = readSettings() || {}
      settings.cloudSync = settings.cloudSync || {}
      settings.cloudSync.lastSyncError = error.message
      writeSettings(settings)
      return { success: false, error: error.message }
    } finally {
      this.isSyncing = false
    }
  }

  /** 从云端拉取并合并配置 */
  async pull(password) {
    if (!this.provider) throw new Error('未配置云存储服务')
    if (this.isSyncing) throw new Error('正在同步中')

    this.isSyncing = true
    try {
      // 列出所有设备文件
      const files = await this.provider.list('devices/')
      const configFiles = files.filter(f => f.name.startsWith('config-') && f.name.endsWith('.json'))

      // 下载并解密所有设备配置
      const remoteConfigs = []
      for (const file of configFiles) {
        const content = await this.provider.download(`devices/${file.name}`)
        const parsed = JSON.parse(content.toString('utf8'))
        if (parsed.version !== 2) continue

        try {
          const decrypted = this.crypto.decryptSyncData(parsed.data, password)
          remoteConfigs.push({
            deviceId: parsed.deviceId,
            deviceName: parsed.deviceName,
            timestamp: parsed.timestamp,
            data: decrypted,
          })
        } catch (err) {
          logger.warn(`[CloudSync] 解密失败 ${file.name}: ${err.message}`)
          // 解密失败可能是密码错误，向上抛出
          if (file.name === `config-${this.deviceId}.json`) {
            throw new Error('SYNC_PASSWORD_INCORRECT')
          }
        }
      }

      // 合并
      const settings = readSettings() || {}
      const merged = this._mergeConfigs(settings, remoteConfigs)
      writeSettings(merged)

      settings.cloudSync = settings.cloudSync || {}
      settings.cloudSync.lastSyncAt = new Date().toISOString()
      settings.cloudSync.lastSyncError = null
      writeSettings(settings)

      logger.info('[CloudSync] Pull succeeded, merged', remoteConfigs.length, 'remote configs')
      return { success: true, mergedFrom: remoteConfigs.map(c => c.deviceName) }
    } catch (error) {
      logger.error('[CloudSync] Pull failed:', error)
      const settings = readSettings() || {}
      settings.cloudSync = settings.cloudSync || {}
      settings.cloudSync.lastSyncError = error.message
      writeSettings(settings)
      return { success: false, error: error.message }
    } finally {
      this.isSyncing = false
    }
  }

  /** 完整同步：先拉取合并，再推送 */
  async sync(password) {
    const pullResult = await this.pull(password)
    if (!pullResult.success) return pullResult
    return this.push(password)
  }

  /** 提取需要同步的数据 */
  _extractSyncData(settings) {
    return {
      apiProfiles: settings.apiProfiles || {},
      currentApiProfile: settings.currentApiProfile || 'default',
      mcpServers: settings.mcpServers || {},
      apiProfilesOrder: settings.apiProfilesOrder || [],
    }
  }

  /** 增量合并策略 */
  _mergeConfigs(localSettings, remoteConfigs) {
    if (remoteConfigs.length === 0) return localSettings

    // 按时间排序，最新的优先
    remoteConfigs.sort((a, b) => new Date(b.timestamp) - new Date(a.timestamp))

    const local = this._extractSyncData(localSettings)

    // 合并 apiProfiles：按 profile 名称合并
    const mergedProfiles = { ...local.apiProfiles }
    for (const remote of remoteConfigs) {
      for (const [name, profile] of Object.entries(remote.data.apiProfiles || {})) {
        if (!mergedProfiles[name]) {
          // 本地没有的 profile，直接加入
          mergedProfiles[name] = profile
        } else {
          // 本地有同名 profile，取更新的版本
          // 策略：如果远端时间戳比本地最后同步时间新，则覆盖
          // 简化处理：取最新远程版本的值
          const remoteTime = new Date(remote.timestamp).getTime()
          const localSyncTime = localSettings.cloudSync?.lastSyncAt
            ? new Date(localSettings.cloudSync.lastSyncAt).getTime()
            : 0
          if (remoteTime > localSyncTime) {
            mergedProfiles[name] = profile
          }
        }
      }
    }

    // 合并 mcpServers：按服务器名称合并
    const mergedServers = { ...local.mcpServers }
    for (const remote of remoteConfigs) {
      for (const [name, server] of Object.entries(remote.data.mcpServers || {})) {
        if (!mergedServers[name]) {
          mergedServers[name] = server
        } else {
          const remoteTime = new Date(remote.timestamp).getTime()
          const localSyncTime = localSettings.cloudSync?.lastSyncAt
            ? new Date(localSettings.cloudSync.lastSyncAt).getTime()
            : 0
          if (remoteTime > localSyncTime) {
            mergedServers[name] = server
          }
        }
      }
    }

    // 合并 apiProfilesOrder：去重保序
    const mergedOrder = [...(local.apiProfilesOrder || [])]
    for (const remote of remoteConfigs) {
      for (const name of (remote.data.apiProfilesOrder || [])) {
        if (!mergedOrder.includes(name)) {
          mergedOrder.push(name)
        }
      }
    }

    // currentApiProfile：取最新远程值
    const latestRemote = remoteConfigs[0]
    const mergedCurrent = latestRemote
      ? latestRemote.data.currentApiProfile || local.currentApiProfile
      : local.currentApiProfile

    // 应用合并结果到本地设置
    localSettings.apiProfiles = mergedProfiles
    localSettings.mcpServers = mergedServers
    localSettings.apiProfilesOrder = mergedOrder
    localSettings.currentApiProfile = mergedCurrent

    return localSettings
  }

  /** 清空云端数据 */
  async clearCloud() {
    if (!this.provider) throw new Error('未配置云存储服务')
    const files = await this.provider.list('devices/')
    for (const file of files) {
      await this.provider.delete(`devices/${file.name}`)
    }
    logger.info('[CloudSync] Cloud data cleared')
  }

  /** 获取已同步设备列表 */
  async getDevices() {
    if (!this.provider) return []
    const files = await this.provider.list('devices/')
    const devices = []
    for (const file of files) {
      if (!file.name.startsWith('config-') || !file.name.endsWith('.json')) continue
      try {
        const content = await this.provider.download(`devices/${file.name}`)
        const parsed = JSON.parse(content.toString('utf8'))
        devices.push({
          deviceId: parsed.deviceId,
          deviceName: parsed.deviceName,
          lastModified: file.lastModified,
          isSelf: parsed.deviceId === this.deviceId,
        })
      } catch {}
    }
    return devices
  }

  /** 移除指定设备的云端数据 */
  async removeDevice(deviceId) {
    if (!this.provider) throw new Error('未配置云存储服务')
    await this.provider.delete(`devices/config-${deviceId}.json`)
    logger.info(`[CloudSync] Removed device: ${deviceId}`)
  }

  _getOrCreateDeviceId() {
    const settings = readSettings() || {}
    if (settings.cloudSync?.deviceId) return settings.cloudSync.deviceId
    const id = crypto.randomUUID()
    settings.cloudSync = settings.cloudSync || {}
    settings.cloudSync.deviceId = id
    writeSettings(settings)
    return id
  }
}

module.exports = SyncService
```

```typescript
import { defineStore } from 'pinia'

export interface CloudSyncStatus {

    enabled: false,

    autoSyncEnabled: false,

    hasPassword: false,

    isAuthorized: false,

    provider: null,

    deviceName: '',

    deviceId: '',

    lastSyncAt: null,

    lastSyncError: null,

    isSyncing: false,

  })

    const result = await window.electronAPI.cloudSyncGetStatus()

    if (result.success) {

      Object.assign(status.value, result)

    }

  }

    if (result.success) status.value.enabled = enabled

  }



  async function setAutoSync(enabled: boolean) {

    const result = await window.electronAPI.cloudSyncSetAutoSync(enabled)

    if (result.success) status.value.autoSyncEnabled = enabled

  }

    if (result.success) status.value.hasPassword = true

    return result

  }

    const result = await window.electronAPI.cloudSyncSyncNow(password)

    if (result.success) {

      status.value.lastSyncAt = new Date().toISOString()

      status.value.lastSyncError = result.error || '同步失败'

    return result

    isConfigured,

    loadStatus,

    toggleEnabled,

    setAutoSync,

    setPassword,

    syncNow,

  }
```

## 十三、同步触发逻辑



### 触发条件矩阵



| 场景 | enabled | autoSync | hasPassword | isAuthorized | 启动时 | 配置变更 | 手动 |

|------|---------|----------|-------------|-------------|--------|---------|------|

| 禁用 | false | - | - | - | - | - | - |

| 手动模式 | true | false | true | true | - | - | ✅ |

| 自动模式 | true | true | true | true | ✅ | ✅（防抖 5s） | ✅ |

| 未配置 | true | - | false/false | - | - | - | 需先配置 |



### 自动同步实现



```javascript

// src/main/services/SyncService.js 中追加



/** 监听配置变更并自动同步 */

startAutoSync(windowsMain) {

  const { fs } = require('fs')

  const { SETTINGS_FILE } = require('./configService')



  // 使用 fs.watch 监听配置文件变更

  let debounceTimer = null

  this._watcher = fs.watch(SETTINGS_FILE, () => {

    if (!this.isSyncing && debounceTimer) {

      clearTimeout(debounceTimer)

    }

    debounceTimer = setTimeout(async () => {

      const settings = readSettings() || {}

      if (settings.cloudSync?.enabled && settings.cloudSync?.autoSyncEnabled) {

        // 需要从渲染进程获取密码（或使用缓存的密码）

        this.emit('auto-sync-triggered')

      }

    }, 5000)

  })

}

```



> **密码缓存问题**：自动同步时需要密码加密数据，但不应持久化密码。

> 方案：在渲染进程中缓存密码（内存中），自动同步时通过 IPC 传递给主进程。

> 应用重启后需重新输入密码才能恢复自动同步。

## 十六、错误处理



| 错误码 | 含义 | 处理方式 |

|--------|------|---------|

| `WEBDAV_AUTH_FAILED` | 认证失败 | 提示检查用户名/密码 |

| `WEBDAV_NOT_FOUND` | 文件/目录不存在 | 首次推送时自动创建目录 |

| `WEBDAV_ERROR_*` | HTTP 错误 | 显示状态码，建议检查网络 |

| `SYNC_PASSWORD_INCORRECT` | 密码错误 | 提示重新输入密码 |

| `ENCRYPTION_FAILED` | 加密失败 | 检查密码格式 |

| `DECRYPTION_FAILED` | 解密失败 | 可能是密码错误或数据损坏 |

| `NETWORK_ERROR` | 网络错误 | 重试 3 次（指数退避），本地继续可用 |
